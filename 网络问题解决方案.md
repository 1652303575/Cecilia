# 网络连接问题解决方案

## 问题现象

如果您遇到以下错误：
```
生成失败: Error generating email reply: Timeout of 60.0s exceeded
503 failed to connect to all addresses
UNKNOWN: ipv4:142.250.73.74:443: tcp handshaker shutdown
```

这表示**无法连接到 Google 服务器**，需要科学上网。

## 解决方案

### 方案 1: 配置代理（推荐）

如果您已经有科学上网工具（如 Clash、V2Ray、ShadowsocksR 等），按以下步骤配置：

#### 1. 查找您的代理端口

**Clash 用户**：
- 打开 Clash
- 查看设置中的 "端口" 或 "Port"
- 通常是 `7890` 或 `7891`

**V2Ray 用户**：
- 打开 V2Ray
- 查看 HTTP 代理端口
- 通常是 `10809` 或 `1080`

#### 2. 配置 .env 文件

编辑项目中的 `.env` 文件（如果没有，从 `.env.example` 复制一个）：

```env
# Google Gemini API Key
GEMINI_API_KEY=你的API密钥

# Server Configuration
HOST=127.0.0.1
PORT=8000

# 代理配置（重要！）
HTTP_PROXY=http://127.0.0.1:7890
HTTPS_PROXY=http://127.0.0.1:7890
```

**注意**：
- 将 `7890` 替换为您的实际代理端口
- 代理地址格式必须是 `http://127.0.0.1:端口号`
- HTTP_PROXY 和 HTTPS_PROXY 都要配置

#### 3. 重启应用

1. 在运行中的终端按 `Ctrl+C` 停止服务器
2. 重新运行 `python main.py` 或 `start.bat`
3. 再次尝试生成回复

### 方案 2: 系统全局代理

如果方案 1 不工作，可以启用系统全局代理：

**Windows**：
1. 确保您的科学上网工具已开启
2. 在工具中启用 "系统代理" 或 "TUN 模式"
3. 重启应用

**macOS/Linux**：
```bash
# 在终端中临时设置代理
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890

# 然后启动应用
python main.py
```

### 方案 3: 测试代理连接

在配置代理后，可以先测试连接：

**Windows PowerShell**：
```powershell
$env:HTTPS_PROXY="http://127.0.0.1:7890"
curl https://www.google.com
```

**Linux/macOS**：
```bash
export HTTPS_PROXY=http://127.0.0.1:7890
curl https://www.google.com
```

如果能看到 HTML 内容，说明代理配置正确。
